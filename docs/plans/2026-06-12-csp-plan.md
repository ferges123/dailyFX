# CSP Implementation Plan

> **For Antigravity:** REQUIRED WORKFLOW: Use `.agent/workflows/execute-plan.md` to execute this plan in single-flow mode.

**Goal:** Add CSP security header to frontend Nginx configuration file.

**Architecture:** Inject Nginx `add_header` directive with Content-Security-Policy in `frontend/nginx.conf`.

**Tech Stack:** Nginx.

---

### Task 1: Add Content-Security-Policy header

**Files:**
- Modify: `frontend/nginx.conf`

**Step 1: Check original file contents**
Review `frontend/nginx.conf` structure to place the header directly inside the `server {}` block.

**Step 2: Add header**
Add the following line right after `server_name _;` or `client_max_body_size 25M;`:
```nginx
  add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob:; connect-src 'self' ws: wss:; frame-ancestors 'none'; object-src 'none'; base-uri 'self';" always;
```

**Step 3: Verification**
Verify the file structure is correct and matches Nginx syntax rules.

**Step 4: Commit**
```bash
git add frontend/nginx.conf
git commit -m "security: add CSP header to Nginx config"
```
